# Contributing to KondoManager

Thanks for considering contributing! Here's how to help:

## How to Contribute

1. **Fork the repository**
   - Click the "Fork" button at the top right of the repository page

2. **Create a new branch**
```bash
git checkout -b feature/YourFeature
```

3. **Make your changes**
   - Write clean, readable code
   - Add comments where necessary
   - Test your changes thoroughly

4. **Commit your changes**
```bash
git commit -m 'Add new feature'
```

5. **Push to your branch**
```bash
git push origin feature/YourFeature
```

6. **Open a Pull Request**
   - Go to the original repository
   - Click "New Pull Request"
   - Select your branch and describe your changes

## Code Style

### PHP
- Follow [PSR-12](https://www.php-fig.org/psr/psr-12/) coding standards
- Use meaningful variable and function names
- Write clear comments for complex logic

### JavaScript/Vue.js
- Follow standard Vue.js formatting conventions
- Use ESLint and Prettier for code formatting
- Keep components small and focused

### General Guidelines
- Write self-documenting code
- Keep functions small and focused on a single task
- Test your code before submitting

## Translations

Help us translate KondoManager into your language!

We use [Crowdin](https://crowdin.com/project/kondomanager) for managing translations:

1. Visit our [Crowdin project page](https://crowdin.com/project/kondomanager)
2. Select your language (or request a new one)
3. Start translating strings
4. Submit your translations for review

Your contributions will help make KondoManager accessible to users worldwide!

## Reporting Issues

If you find a bug or have a suggestion:
1. Check if the issue already exists
2. Create a new issue with a clear title and description
3. Include steps to reproduce (for bugs)
4. Add relevant screenshots if applicable

## Pull Request Guidelines

- Keep PRs focused on a single feature or fix
- Update documentation if needed
- Reference related issues in your PR description
- Be responsive to feedback and comments
